2) Conflict Recovery Sequence

sequenceDiagram
    participant A as Android Runner
    participant API as Laravel API
    participant R as GBCR/RCLR Resolver
    participant DB as SQLite
    participant W as Web Architect

    A->>API: /sessions/{id}/state (old version in session)
    W->>API: Publish new survey version
    API->>R: Reconcile(session, activeVersion)
    R->>DB: Load answers + new schema graph
    alt replay path still valid
        R-->>API: atomic_recovery + continue_from=current
    else replay invalid but stable fallback found
        R-->>API: rollback + fallback_node + dropped_answers
    else no stable fallback node
        R-->>API: rollback(restart_from_entry)
    end
    API->>DB: Persist session state + conflict log
    API-->>A: state + version_sync payload
    A->>A: Render reachable node only (no zombie question)

6) State Diagram: Survey Session Conflict Lifecycle

stateDiagram-v2
    [*] --> InProgress

    InProgress --> SyncCheck: action request

    SyncCheck --> InProgress
    SyncCheck --> AtomicRecovery
    SyncCheck --> RollbackToFallback
    SyncCheck --> NuclearRestart

    AtomicRecovery --> ReachabilityGuard
    RollbackToFallback --> ReachabilityGuard
    NuclearRestart --> ReachabilityGuard

    ReachabilityGuard --> InProgress: return reconciled state
    InProgress --> Completed: complete and terminal path reached
    Completed --> [*]

    note right of SyncCheck
        Decision rules:
        1) no mismatch -> InProgress
        2) mismatch + replay valid -> AtomicRecovery
        3) mismatch + fallback exists -> RollbackToFallback
        4) mismatch + no fallback -> NuclearRestart
    end note

    note right of RollbackToFallback
        Drop answers at and after fallback boundary.
        Fallback node must be answered fresh.
    end note
